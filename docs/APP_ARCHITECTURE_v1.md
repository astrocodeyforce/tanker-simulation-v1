# APPLICATION ARCHITECTURE

## Overview

The application is a **config-driven, headless simulation pipeline** with four components:

```
┌──────────────┐      ┌───────────────────┐      ┌──────────────────┐
│  YAML Config │─────►│  Scenario Runner   │─────►│  Run Output Dir  │
│  (inputs)    │      │  (run_scenario.sh) │      │  data/runs/<ts>/ │
└──────────────┘      └────────┬──────────┘      └──────┬───────────┘
                               │                        │
                    ┌──────────┴──────────┐    ┌────────┴──────────┐
                    │  OpenModelica       │    │  Python Plotter   │
                    │  (TankerTransfer.mo)│    │  (plot_results.py)│
                    │  Compile → Simulate │    │  CSV → PNG plots  │
                    │  → Export CSV       │    │                   │
                    └─────────────────────┘    └───────────────────┘
```

---

## Component A: Modelica Core Model (Physics)

**File:** `modelica/TankerTransfer.mo`

### Submodels

| Block | Responsibility |
|-------|---------------|
| **Ullage Gas** | Ideal gas law: P·V = n·R·T. Tracks gas mass, pressure, volume as liquid leaves. |
| **Liquid Inventory** | Mass/volume balance: `dV_liquid/dt = -Q_out`. Tracks what's left in the tanker. |
| **Hose Pressure Drop** | Darcy-Weisbach friction + minor losses. Auto-switches laminar/turbulent via Reynolds number. |
| **Backpressure Boundary** | Receiver pressure as a constant boundary condition. |
| **Pressure Limit Controller** | When `P_tank >= max_pressure`, air inflow drops to zero. Prevents over-pressurization. |
| **Air Supply** | Splits incoming compressed air between tank ullage and pump based on config fractions. |
| **AODD Pump (simplified)** | `Q_pump = efficiency * mdot_air_to_pump`. Placeholder until manufacturer curve added. |

### Model Parameters (Set From Config)

All physics parameters are passed as Modelica `parameter` values. The scenario runner reads the YAML config and injects them as `-override` flags to the `omc` simulation command.

## Component B: Scenario Runner

**File:** `scripts/run_scenario.sh`

### Workflow

```
1. Read YAML config file (using Python helper for parsing)
2. Create timestamped run directory: data/runs/<timestamp>_<scenario>/
3. Copy input config to run directory as inputs.yaml
4. Convert YAML parameters → Modelica -override string
5. Compile TankerTransfer.mo with omc
6. Run simulation with overrides
7. Export results to CSV (using export_results.py for MAT→CSV if needed)
8. Write run_log.json
```

### Parameter Injection

```bash
# Example: YAML config → Modelica override string
-override=V_tank_total=17.034,V_liquid_0=15.33,P_max=308168,...
```

All unit conversions (gallons→m³, psig→Pa, cP→Pa·s, etc.) happen in the runner before passing to Modelica.

## Component C: Visualization Module

**File:** `python/plot_results.py`

### Generated Plots (single PNG with 4 subplots)

| Subplot | X-axis | Y-axis | Description |
|---------|--------|--------|-------------|
| Top-left | Time (min) | P_tank (psig) | Tank ullage pressure over time |
| Top-right | Time (min) | Q_out (GPM) | Instantaneous liquid flow rate |
| Bottom-left | Time (min) | V_remaining (gal) | Liquid still in the tanker |
| Bottom-right | Time (min) | V_transferred (gal) | Cumulative liquid delivered |

### Input/Output

- **Input:** `data/runs/<run_dir>/outputs.csv`
- **Output:** `data/runs/<run_dir>/plots.png`

## Execution Flow

### Single Scenario

```bash
cd /opt/sim-lab/truck-tanker-sim-env
docker compose run --rm openmodelica-runner bash /work/scripts/run_scenario.sh \
    /work/config/scenario_A_pressurize_only.yaml
docker compose run --rm python-plotter python /work/python/plot_results.py \
    /work/data/runs/<latest>/outputs.csv
```

### All Scenarios + Guard Check

```bash
cd /opt/sim-lab/truck-tanker-sim-env
./scripts/run_app.sh
```

This runs:
1. Guard snapshot
2. Scenario A → simulate → plot
3. Scenario B → simulate → plot
4. Scenario C → simulate → plot
5. Guard verify
6. Log summary

## Component E: Visualization Dashboard Layer

**File:** `python/dashboard.py`

### Responsibilities

| Responsibility | Description |
|---------------|-------------|
| **Scan simulation outputs** | Discover all run directories under `data/runs/` |
| **Load CSV result files** | Read `outputs.csv` and `run_log.json` from each run |
| **Render Plotly interactive charts** | Pressure, flow, volume, and transferred curves with zoom/pan |
| **Provide browser-based UI** | Streamlit app accessible via SSH tunnel on port 8501 |

### Separation of Concerns

```
┌─────────────────────────┐      ┌─────────────────────────────┐
│   SIMULATION ENGINE     │      │   VISUALIZATION ENGINE      │
│                         │      │                             │
│  OpenModelica (omc)     │─CSV─►│  Streamlit + Plotly         │
│  run_scenario.sh        │      │  dashboard.py               │
│  export_results.py      │      │                             │
│                         │      │  Reads data/runs/ (read)    │
│  Writes data/runs/ (rw) │      │  Serves on 127.0.0.1:8501  │
└─────────────────────────┘      └─────────────────────────────┘
```

- The simulation engine **writes** results — it has no knowledge of the dashboard
- The visualization engine **reads** results — it never modifies simulation data
- They share the `data/` bind mount but have no runtime coupling
- The dashboard container runs independently via `docker compose up -d visual-dashboard`

### Service Configuration

| Property | Value |
|----------|-------|
| Docker service | `visual-dashboard` |
| Container name | `simlab-dashboard` |
| Image | `python:3.12-slim` |
| Port binding | `127.0.0.1:8501:8501` (localhost only) |
| Dependencies | `requirements.txt` (streamlit, plotly, pandas) |
| Resource limits | CPU: 1.0, Memory: 1 GB |
